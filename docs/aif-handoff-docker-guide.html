01
Требования
Что нужно установить перед началом
🐳
Docker Desktop
Docker Desktop для Windows/macOS или Docker Engine + Docker Compose для Linux. Версия с поддержкой compose v2.
🔑
Claude аккаунт
Подписка Claude (авторизация через claude login внутри контейнера) или Anthropic API Key.
🌿
Git
Для клонирования репозитория.
🌐
Домен (для production)
Только для production-деплоя с SSL. Для локального dev не нужен.
Преимущество Docker: Не нужно устанавливать Node.js или Claude CLI на хост-машину — всё изолировано внутри контейнеров. Данные сохраняются в Docker volumes и переживают перезапуски.
02
Клонирование репозитория
Получить исходный код aif-handoff
bash 
$ git clone https://github.com/lee-to/aif-handoff.git
$ cd aif-handoff

Структура репозитория после клонирования:

tree
aif-handoff/
├── packages/
│   ├── api/        # Hono REST + WebSocket  → :3009
│   ├── web/        # React Kanban UI        → :5180
│   └── agent/      # Координатор субагентов
├── docker-compose.yml             # dev режим
├── docker-compose.production.yml  # production
├── .env.example
└── package.json
03
Быстрый старт (Dev режим)
Одна команда собирает и запускает все три сервиса
bash 
$ docker compose up --build

Эта команда собирает образы и запускает три сервиса:

СервисURLОписание
API http://localhost:3009 Hono REST + WebSocket сервер
Web UI http://localhost:5180 React Kanban-доска (Angie reverse proxy)
Agent background Координатор субагентов, polling каждые 30 сек
Reverse proxy: Web UI на localhost:5180 автоматически проксирует все API и WebSocket запросы через Angie — дополнительная настройка не нужна.

Запуск в фоновом режиме

bash 
$ docker compose up --build -d   # -d = detached (фон)
$ docker compose logs -f          # просмотр логов
04
Авторизация Claude
Два способа — API Key или подписка Claude через OAuth
Option A
🔑 API Key
Создайте файл .env с API ключом до запуска контейнеров. Подходит для CI/CD и серверного деплоя.
Option B
👤 Claude подписка
Войдите через OAuth внутри контейнера после первого запуска. Credentials сохраняются в Docker volume.

Option A — API Key

Создайте .env из примера и задайте ключ до запуска:

bash 
$ cp .env.example .env
.env 
ANTHROPIC_API_KEY=sk-ant-xxxxx

Option B — Claude подписка (OAuth)

1
Запустить контейнеры
bash
$ docker compose up --build -d
2
Войти в контейнер агента
bash
$ docker compose exec agent claude login
3
Открыть URL в браузере
Терминал выведет URL для авторизации. Скопируйте его полностью и откройте в браузере.
Важно: Терминал может переносить длинный URL на несколько строк — удалите все переносы строк и пробелы перед вставкой, иначе OAuth завершится ошибкой invalid code_challenge.
4
Перезапустить контейнеры
bash
$ docker compose restart

Credentials сохраняются в Docker volume claude-auth и переживают все перезапуски.

05
Переменные окружения (Dev)
Docker-специфичные переменные для dev-режима
ПеременнаяПо умолчаниюОписание
ANTHROPIC_API_KEY — API ключ. Если пусто — используется авторизация через claude login
PORT 3009 Порт API-сервера на хосте
WEB_PORT 5180 Порт Web UI на хосте
PROJECTS_DIR ./projects Директория на хосте для файлов проектов (bind mount)
PROJECTS_MOUNT /home/www Путь к файлам проектов внутри контейнеров
AGENT_USE_SUBAGENTS true true — субагенты (качество), false — AIF-скиллы (скорость)
AGENT_BYPASS_PERMISSIONS true Разрешить агенту выполнять команды без подтверждения
POLL_INTERVAL_MS 30000 Интервал опроса задач координатором (мс)
LOG_LEVEL debug Уровень логов: fatal / error / warn / info / debug / trace

Пример минимального .env для Docker Dev

.env 
# Авторизация (один из вариантов)
# ANTHROPIC_API_KEY=sk-ant-xxxxx

# Порты
PORT=3009
WEB_PORT=5180

# Проекты — путь к вашему коду на хосте
PROJECTS_DIR=./projects
PROJECTS_MOUNT=/home/www

# Агент
AGENT_BYPASS_PERMISSIONS=true
POLL_INTERVAL_MS=30000
06
Проверка работоспособности
Убедитесь что все сервисы запущены корректно
1
Открыть Kanban UI
Перейдите на http://localhost:5180 — должна появиться Kanban-доска с колонками Backlog, Planning, Plan Ready, Implementing.
2
Проверить readiness агента
bash
$ curl -s http://localhost:3009/agent/readiness
3
Проверить статус контейнеров
bash
$ docker compose ps

Все три сервиса (api, web, agent) должны иметь статус running.

4
Создать проект и добавить задачу
В UI создайте проект, укажите путь к коду (внутри контейнера это /home/www/ваш-проект). Добавьте задачу в Backlog — через ~30 сек агент начнёт её обрабатывать.
Признак успешного запуска: В логах агента видны строки "Agent coordinator is running" и "Poll cycle complete" каждые 30 секунд.
07
Production запуск
Отдельный compose-файл с усиленной безопасностью и healthchecks
Production включает: только порты 80/443, security hardening, healthchecks, resource limits и log rotation. Авторизация работает так же как в dev.
bash 
$ docker compose -f docker-compose.production.yml up --build

В фоновом режиме:

bash 
$ docker compose -f docker-compose.production.yml up --build -d

Авторизация в production

Работает так же как в dev — два варианта:

bash — Option B (подписка) 
$ docker compose -f docker-compose.production.yml exec agent claude login
$ docker compose -f docker-compose.production.yml restart
08
Production переменные окружения
Дополнительные переменные специфичные для production
ПеременнаяПо умолчаниюОписание
DOMAIN localhost Домен для SSL сертификата (Let's Encrypt через ACME)
HTTP_PORT 80 Порт HTTP для Web UI (production)
HTTPS_PORT 443 Порт HTTPS (production)
ANTHROPIC_API_KEY — API ключ (или использовать claude login)
PROJECTS_DIR ./projects Путь к проектам на хосте
PROJECTS_MOUNT /home/www Путь внутри контейнеров

Пример .env для production

.env 
# Авторизация
ANTHROPIC_API_KEY=sk-ant-xxxxx   # или оставить пустым + claude login

# Домен и SSL
DOMAIN=handoff.example.com

# Production порты
HTTP_PORT=80
HTTPS_PORT=443

# Проекты
PROJECTS_DIR=/var/projects
PROJECTS_MOUNT=/home/www

# Агент
AGENT_BYPASS_PERMISSIONS=true
LOG_LEVEL=info   # в production лучше info вместо debug
09
SSL и домен
Автоматический SSL через Let's Encrypt (ACME)
1
Укажите ваш домен в DOMAIN=handoff.example.com в .env
2
Убедитесь что DNS A-запись домена указывает на IP сервера
3
Порты 80 и 443 должны быть открыты в firewall сервера
4
Запустите production compose — Angie автоматически получит и обновит сертификат через ACME
✓
Сертификат хранится в Docker volume и переживает перезапуски контейнеров
Локальная разработка: Для dev оставьте DOMAIN=localhost — SSL не нужен, приложение доступно по HTTP на порту 5180.

Полезные команды Docker
Управление контейнерами в процессе работы

$ docker compose down

# Статус контейнеров
$ docker compose ps

# Логи всех сервисов
$ docker compose logs -f

# Логи конкретного сервиса
$ docker compose logs -f agent

# Войти в контейнер агента
$ docker compose exec agent sh

# Пересобрать без кэша
$ docker compose build --no-cache
11
Volumes и персистентность данных
Что сохраняется между перезапусками контейнеров
VolumeСодержимое
claude-auth Credentials от claude login — переживает все перезапуски
aif-data SQLite база данных с задачами, проектами, историей
projects (bind mount) Файлы проектов с хост-машины — монтируется из PROJECTS_DIR

Подключение вашего проекта

Чтобы агент работал с кодом на вашей машине, укажите путь в .env:

.env 
# Linux / macOS
PROJECTS_DIR=/home/user/my-project
PROJECTS_MOUNT=/home/www

# Windows
PROJECTS_DIR=C:/automost
PROJECTS_MOUNT=/home/www

В UI при создании проекта укажите Root Path = /home/www (путь внутри контейнера).

Удаление volumes: При docker compose down -v все volumes удаляются включая базу данных и credentials. Используйте просто docker compose down для остановки без удаления данных.
12
Устранение проблем
Типичные ошибки при работе с Docker
!
Контейнер не запускается — порт занят
Измените порты в .env:
.env
PORT=3010
WEB_PORT=5181
!
Агент не видит credentials после claude login
Обязательно перезапустите контейнеры после авторизации:
bash
$ docker compose restart
!
OAuth ошибка invalid code_challenge
URL из терминала перенёсся на несколько строк. Скопируйте его целиком, удалите все переносы строк и пробелы, затем вставьте в браузер.
!
Агент зависает или таймаутится
Отключите субагентов в .env:
.env
AGENT_USE_SUBAGENTS=false
Перезапустите: docker compose restart
!
Агент сообщает об отсутствии прав
Проверьте что в .env установлено:
.env
AGENT_BYPASS_PERMISSIONS=true
!
Полный сброс и чистый старт
bash
$ docker compose down          # остановить (данные сохранятся)
$ docker compose down -v        # остановить + удалить volumes (данные потеряются)
$ docker compose build --no-cache # пересобрать образы
$ docker compose up --build     # запустить заново
Всё готово! Web UI на :5180 · API на :3009 · Agent в контейнере · Данные в Docker volumes. Добавляйте задачи в Backlog и наблюдайте как агент проводит их через все стадии автономно.
