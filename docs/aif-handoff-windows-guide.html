01
Системные требования
Что должно быть установлено перед началом
⬢
Node.js 22+
Необходим для запуска всех сервисов. Рекомендуется установка через nvm-windows.
🤖
Claude Code CLI
Agent SDK запускает Claude Code как дочерний процесс — CLI обязателен глобально.
🔑
Claude аккаунт
Подписка Claude или Anthropic API Key. Credentials хранятся в ~/.claude/
🌿
Git
Для клонирования репозитория. Агент также использует git для работы с кодом проекта.
02
Проверка Claude CLI на Windows
PowerShell-аналоги unix-команд — which и grep не работают в Windows
Важно: На Windows команды which и grep не работают. Используйте PowerShell-аналоги ниже.

Проверить установку и версию

powershell 
PS> claude --version
# Ожидаемый вывод:
2.1.77 (Claude Code)

PS> Get-Command claude
# Покажет путь к claude.ps1
ExternalScript  claude.ps1  C:\Users\...\AppData\Local\nvm\nodejs\claude.ps1

Проверить глобальный npm-пакет

powershell 
PS> npm list -g --depth=0 | Select-String "claude-code"
# Ожидаемый вывод:
+-- @anthropic-ai/claude-code@2.1.77

Проверить авторизацию

powershell 
PS> ls $env:USERPROFILE\.claude\
# Должны быть файл .credentials.json и папки sessions/, cache/ и др.

PS> Test-Path "$env:USERPROFILE\.claude\.credentials.json"
True  # OK — авторизован
False # Нужна авторизация: claude login
Если не авторизован: Запустите claude login, скопируйте URL из терминала целиком без переносов строк и вставьте в браузер. Переносы строк в URL ломают OAuth.

Если CLI не установлен

powershell 
PS> npm install -g @anthropic-ai/claude-code
PS> claude --version  # проверить после установки
03
Клонирование репозитория
aif-handoff должен располагаться отдельно от вашего рабочего проекта
Архитектура: aif-handoff — отдельный сервис (монорепозиторий с API, Web UI и Agent). Не клонируйте его внутрь вашего проекта. Рекомендуется C:\aif-handoff.
powershell 
PS> cd C:\
PS> git clone https://github.com/lee-to/aif-handoff.git
PS> cd aif-handoff

Структура монорепозитория после клонирования:

tree
C:\aif-handoff\
├── packages/
│   ├── api/       # Hono REST + WebSocket  → :3009
│   ├── web/       # React Kanban UI        → :5180
│   ├── agent/     # Координатор субагентов
│   ├── shared/    # Типы, схема, state machine
│   └── data/      # Централизованный DB-слой
├── .env.example
├── .ai-factory.json
└── package.json

06
Конфигурация окружения
Создайте .env из примера и настройте ключевые переменные
powershell 
PS C:\aif-handoff> copy .env.example .env
PS C:\aif-handoff> notepad .env
Переменная Значение Описание
ANTHROPIC_API_KEY пусто Оставить пустым если есть ~/.claude/.credentials.json (claude login)
PORT 3009 Порт API-сервера
WEB_PORT 5180 Порт Web UI (Kanban-доска)
AGENT_BYPASS_PERMISSIONS true Уже выставлено в .env.example — разрешает агенту выполнять команды
AGENT_WAKE_ENABLED false Windows! Отключить WebSocket wake-channel — на Windows вызывает ошибку WebSocket is not defined
AGENT_USE_SUBAGENTS true Закомментировано — default true. Поставить false для ускорения (без итеративного уточнения)
DATABASE_URL ./data/aif.sqlite Путь к SQLite-файлу
POLL_INTERVAL_MS 30000 Интервал опроса задач координатором (мс)
Windows-специфично: Обязательно добавьте AGENT_WAKE_ENABLED=false — иначе агент будет спамить ошибкой WebSocket is not defined при попытках подключения. Polling-режим работает полноценно.

Итоговый минимальный .env для Windows:

.env 
# Порты
PORT=3009
WEB_PORT=5180
DATABASE_URL=./data/aif.sqlite

# Агент
AGENT_BYPASS_PERMISSIONS=true
AGENT_WAKE_ENABLED=false   # ВАЖНО для Windows!
POLL_INTERVAL_MS=30000

# Авторизация (оставить пустым если используется claude login)
# ANTHROPIC_API_KEY=sk-ant-xxxxx

# Логирование
LOG_LEVEL=debug
AGENT_QUERY_AUDIT_ENABLED=false
07
Интеграция с существующим aif-factory
Как связать aif-handoff с проектом, где уже инициализирован AI Factory
Принцип: aif-handoff — сервис-оркестратор. Он запускает агентов из .claude/agents/*.md вашего рабочего проекта. aif-factory предоставляет агентов, aif-handoff — управляет ими автономно.

Типичная структура до интеграции:

tree
C:\automost\               # ваш рабочий проект
├── .ai-factory/           # рабочие файлы ai-factory
├── .claude/
│   ├── skills/            # скиллы (/ai-factory.task, /ai-factory.feature и др.)
│   └── agents/            # ← агенты, которые подхватывает aif-handoff
├── .ai-factory.json
├── back/                   # бэкенд
└── front/                  # фронтенд

C:\aif-handoff\             # aif-handoff (отдельно!)
├── packages/
└── data/aif.sqlite        # база задач

Что нужно проверить в вашем проекте

✓
Наличие папки C:\automost\.claude\agents\ с определениями агентов
✓
Наличие файла C:\automost\.ai-factory.json — конфиг AI Factory
i
Если агентов нет — aif-handoff сам создаст стандартные при инициализации проекта через UI

Связь через Root Path в UI

При создании проекта в Kanban UI укажите Root Path = C:\automost. Агент будет работать именно с этой директорией — читать код, писать файлы, делать коммиты.

Создание проекта в UI
Откройте http://localhost:5180 и создайте первый проект
1
Открыть Kanban UI
Перейдите на http://localhost:5180 — должна появиться пустая Kanban-доска с колонками Backlog, Planning, Plan Ready, Implementing.
2
Нажать на селектор проекта (верхний левый угол)
Выбрать "Create Project" из выпадающего меню.
3
Заполнить форму проекта
Name: Automost (или любое название)
Root Path: C:\automost — абсолютный путь к вашему проекту

Root Path — это директория где агент будет создавать файлы, читать код и делать коммиты. Укажите корень вашего рабочего проекта.
4
Runtime Profiles — оставить по умолчанию
Значение (none) в Runtime Profiles означает использование встроенного Claude runtime через ~/.claude/credentials.json. Дополнительная настройка не нужна если авторизация выполнена через claude login.
10
Добавление задач
Как правильно формулировать задачи для AI-агентов

Нажмите "+ Add task" в колонке Backlog. Задача должна быть конкретной и самодостаточной — агент прочитает её и составит план реализации.

✅
Хорошая задача
Добавить endpoint GET /api/users/{id} который возвращает данные пользователя из таблицы users. Если пользователь не найден — вернуть 404 с JSON-ошибкой.
❌
Плохая задача
Улучшить сайт. — Слишком размыто, агент не знает что именно делать.

Что происходит после добавления

→
Backlog → Planning
Через ~30 сек агент подхватывает задачу и запускает plan-coordinator — итеративное составление плана реализации.
→
Planning → Plan Ready
Агент завершил составление плана. Можно одобрить или запросить изменения. В auto-режиме — переходит автоматически.
→
Plan Ready → Implementing
Запускается implement-coordinator — параллельное выполнение задач плана с quality sidecars.
→
Implementing → Review → Done
Параллельно запускаются review-sidecar и security-sidecar. При обнаружении проблем — автоматический возврат в Implementing.
11
Windows-специфичные нюансы
Особенности запуска без WSL2 и Docker
WebSocket is not defined: На Windows Node.js может не иметь глобального WebSocket. Решение — AGENT_WAKE_ENABLED=false в .env. Агент переключится на polling-only режим (каждые 30 сек) — функционально полноценно.
Unix-командаPowerShell-аналог
which claude Get-Command claude
npm list -g | grep claude npm list -g --depth=0 | Select-String "claude-code"
ls ~/.claude/ ls $env:USERPROFILE\.claude\
cp .env.example .env copy .env.example .env
cat .env notepad .env или Get-Content .env
claude.ps1: На Windows Claude Code устанавливается как claude.ps1 скрипт через nvm. Это нормально — claude --version работает также как в Unix.
12
Устранение проблем
Типичные ошибки и способы их решения
!
Агент зависает или часто таймаутится
Отключите субагентов для ускорения в .env:
.env
AGENT_USE_SUBAGENTS=false
Это переключит на однопроходные AIF-скиллы — быстрее, без итеративного уточнения.
!
Агент сообщает об отсутствии прав
Уже решено через AGENT_BYPASS_PERMISSIONS=true в .env. Если всё равно возникает — проверьте что файл .env сохранён и перезапустите npm run dev.
!
Missing script: "aif:init"
В текущей версии скрипт называется init, не aif:init:
powershell
PS> npm run init     # ✓ правильно
PS> npm run aif:init # ✗ не существует
!
Задача застряла в одной стадии
Агент имеет watchdog — если стадия не обновляется дольше AGENT_STAGE_STALE_TIMEOUT_MS (90 мин по умолчанию), задача переходит в blocked_external и автоматически перезапускается (до 3 раз). После 3 попыток — quarantine, нужно ручное вмешательство через UI.
!
Порты уже заняты
Если 3009 или 5180 заняты другими процессами — измените в .env:
.env
PORT=3010
WEB_PORT=5181
Всё готово! API на :3009 · Web UI на :5180 · Agent в polling-режиме · Проект C:\automost подключён. Добавляйте задачи в Backlog и наблюдайте как агент проводит их через все стадии автономно.
