AIF Handoff
Dev Container Guide

Запуск AIF Handoff в изолированном Dev Container прямо из VS Code или JetBrains IDE. Node 22, все зависимости и проброшенные порты — предустановлены автоматически. Не нужно ничего устанавливать на хост.

🐳 Docker Desktop
💙 VS Code
🟣 JetBrains IDE
🔑 Claude аккаунт
💙 VS Code
🟣 JetBrains (IntelliJ, WebStorm, PyCharm...)
01
Что такое Dev Container
Изолированная среда разработки внутри Docker-контейнера, управляемая из IDE
Принцип: Dev Container — это Docker-контейнер со всем необходимым окружением (Node.js 22, зависимости, порты), которым управляет ваша IDE. Открываете проект — IDE предлагает переоткрыть его в контейнере.
⚡
Нет ручной настройки
Node.js 22, npm, все зависимости предустановлены автоматически при открытии контейнера.
🔒
Полная изоляция
Среда не конфликтует с другими проектами и системными зависимостями на хосте.
🌐
Порты пробрасываются
Порты 3009 (API) и 5180 (Web UI) автоматически пробрасываются на хост.
💾
Код остаётся на хосте
Файлы проекта монтируются из хоста — редактируете как обычно, изменения сразу видны.

Конфигурация Dev Container находится в репозитории:

tree
aif-handoff/
└── .devcontainer/
    └── devcontainer.json   # конфигурация контейнера

Требования
Что нужно установить перед началом
💙
VS Code
  • VS Code (последняя версия)

  • Docker Desktop
  • Git
🟣
JetBrains
  • IntelliJ IDEA, WebStorm, PyCharm или другая JetBrains IDE

Docker Desktop обязателен: Dev Container работает поверх Docker. Убедитесь что Docker Desktop запущен перед открытием проекта в IDE.
03
Клонирование репозитория
Получить исходный код на хост-машину
bash 
$ git clone https://github.com/lee-to/aif-handoff.git
$ cd aif-handoff
Или клонировать прямо из IDE: VS Code и JetBrains поддерживают клонирование через встроенный Git-клиент — Git: Clone в командной строке VS Code или File → New → Project from VCS в JetBrains.
04
VS Code — установка расширения
Установить Dev Containers extension от Microsoft
1
Открыть панель расширений
В VS Code нажмите Ctrl+Shift+X (или Cmd+Shift+X на macOS) для открытия Extensions.
2
Найти и установить Dev Containers
Введите в поиске: Dev Containers
Установите расширение от Microsoft (ID: ms-vscode-remote.remote-containers)
3
Убедиться что Docker Desktop запущен
Dev Containers требует запущенный Docker. Иконка Docker в трее должна показывать "Docker Desktop is running".
bash — альтернативно через CLI 
$ code --install-extension ms-vscode-remote.remote-containers
05
VS Code — открытие в контейнере
Два способа открыть проект в Dev Container

Способ 1 — через уведомление (рекомендуется)

1
Открыть папку проекта
File → Open Folder → выбрать папку aif-handoff
2
Принять предложение VS Code
VS Code автоматически обнаружит .devcontainer/devcontainer.json и покажет уведомление в правом нижнем углу:

Reopen in Container

Нажмите эту кнопку.
3
Дождаться сборки контейнера
VS Code соберёт образ и запустит контейнер. При первом запуске это займёт несколько минут. Прогресс виден в уведомлении — нажмите "Show log" для деталей.

Способ 2 — через Command Palette

vs code — command palette
# Открыть Command Palette
Ctrl+Shift+P  (или Cmd+Shift+P на macOS)

# Ввести команду
Dev Containers: Reopen in Container
После открытия: В левом нижнем углу VS Code появится индикатор Dev Container: aif-handoff — это означает что вы работаете внутри контейнера. Терминал VS Code теперь работает внутри контейнера.
06
JetBrains — настройка
Установить плагин Dev Containers для JetBrains IDE
1
Открыть настройки плагинов
File → Settings → Plugins (Windows/Linux)
IntelliJ IDEA → Preferences → Plugins (macOS)
2
Найти и установить Dev Containers
На вкладке Marketplace найдите Dev Containers и нажмите Install.
После установки перезапустите IDE.
3
Настроить Docker в IDE
Settings → Build, Execution, Deployment → Docker
Нажмите + и выберите Docker Desktop (или Unix socket на Linux). IDE должна показать Connection successful.
07
JetBrains — открытие в контейнере
Открыть проект aif-handoff в Dev Container
1
Открыть проект
File → Open → выбрать папку aif-handoff
2
Принять предложение IDE
JetBrains обнаружит .devcontainer/devcontainer.json и покажет уведомление:

Reopen in Dev Container

Нажмите эту кнопку.
3
Дождаться запуска контейнера
IDE соберёт и запустит контейнер. Прогресс отображается в нижней панели статуса. При первом запуске займёт несколько минут.
Альтернативно: В JetBrains можно открыть Dev Container через File → Remote Development → Dev Containers → Open Existing и указать путь к папке с .devcontainer.
08
Авторизация Claude
Настроить доступ к Claude внутри Dev Container

Два варианта — выберите один:

Option A — API Key (рекомендуется для Dev Container)

Создайте .env из примера до открытия контейнера:

bash 
$ cp .env.example .env
.env 
ANTHROPIC_API_KEY=sk-ant-xxxxx

Option B — Claude подписка (OAuth внутри терминала)

Откройте терминал внутри контейнера (в VS Code — Ctrl+`, в JetBrains — вкладка Terminal) и выполните:

bash — терминал внутри контейнера 
$ claude login
URL из терминала: Скопируйте URL целиком без переносов строк и откройте в браузере на хосте. Переносы строк ломают OAuth с ошибкой invalid code_challenge.
09
Конфигурация .env
Настройка переменных окружения для Dev Container
bash — создать из примера 
$ cp .env.example .env
ПеременнаяПо умолчаниюОписание
ANTHROPIC_API_KEY — Оставить пустым если используется claude login
PORT 3009 Порт API — пробрасывается на хост автоматически
WEB_PORT 5180 Порт Web UI — пробрасывается на хост автоматически
AGENT_BYPASS_PERMISSIONS true Разрешить агенту выполнять команды
AGENT_USE_SUBAGENTS true false для ускорения на слабых машинах
POLL_INTERVAL_MS 30000 Интервал опроса задач (мс)
10
Запуск и проверка
Запустить сервисы из терминала внутри контейнера
Важно: Все команды выполняются в терминале внутри контейнера (терминал IDE после открытия в Dev Container), не на хосте.
1
Установить зависимости
bash
$ npm install

В Dev Container зависимости могут устанавливаться автоматически — проверьте лог запуска контейнера.

2
Инициализировать базу данных
bash
$ npm run init
3
Запустить все сервисы
bash
$ npm run dev
4
Открыть Web UI
Перейдите на http://localhost:5180 в браузере на хосте — порт пробрасывается автоматически.

VS Code также покажет уведомление с кнопкой "Open in Browser" при старте сервера.
Готово: API на :3009 · Web UI на :5180 · Agent работает внутри контейнера. Создайте проект в UI и добавьте задачу в Backlog.
11
Устранение проблем
Типичные проблемы Dev Container
!
IDE не предлагает "Reopen in Container"
Убедитесь что расширение/плагин Dev Containers установлен и Docker Desktop запущен. В VS Code вызовите вручную:
Ctrl+Shift+P → Dev Containers: Reopen in Container
!
Контейнер не собирается — ошибки Docker
Проверьте что Docker Desktop запущен и имеет достаточно ресурсов (минимум 4GB RAM). Попробуйте пересобрать:
vs code — command palette
Dev Containers: Rebuild Container
!
Порты не пробрасываются
В VS Code проверьте вкладку Ports (рядом с Terminal). Если портов нет — добавьте вручную: нажмите "Forward a Port" и введите 3009 и 5180.
!
Агент зависает или нет прав
Добавьте в .env внутри контейнера:
.env
AGENT_BYPASS_PERMISSIONS=true
AGENT_USE_SUBAGENTS=false  # если медленно работает
Перезапустите npm run dev.
